# Otus - 5. Дисковая подсистема 
## Изучение рейд массивов и сборка Raid 10
Данный тестовый стенд был собран под виртуализацию на Virtual Box и использовался образ centos 7.

Цели задания:

1. добавить в Vagrantfile еще дисков.
2. собрать R0/R5/R10 на выбор.
3. прописать собранный рейд в конф, чтобы рейд собирался при загрузке.
4. сломать/починить raid.
5. создать GPT раздел и 5 партиций.

### 1. Добалвение в Vagrantfile диска 

```
:sata5 => {
                        :dfile => './VirtualBox VMs/5Raid_otuslinux_1711111286616_46482/sata5.vdi',
                        :size => 250, # Megabytes
                        :port => 5
                }
```
### 2. Cобрать R0/R5/R10 на выбор.
На тестовом стенде настроил R10 через утилиту mdadm.
Перед настройка рейда необходимо убедиться что все диски определяются в системе, это можно увидеть через команду

```
fstab -l
```

Для создания рейда выполняем команду с опцией -l уровень RAID и опцией -n указывающие кол-во устройств.

```
mdadm --create --verbose /dev/md0 -l 10 -n 5 /dev/sd{b,c,d,e,f}
``` 

Далее проверяем что рейд собрался корректно 

```
cat /proc/mdstat
```

### 3. прописать собранный рейд в конф, чтобы рейд собирался при загрузке.

После проверки что raid собрался корретно и все диски подключены создаем конф файл в каталоге /etc/mdadm.conf

Затем двумя коммандами заполняем конф файл mdadm.conf

```
echo "DEVICE partitions" > /etc/mdadm.conf
mdadm --detail --scan --verbose | awk '/ARRAY/ {print}' >>/etc/mdadm.conf
```

### 4. Cломать/починить raid.

Искусственно воссоздали поломку одного из дисков командой.
```
mdadm /dev/md0 --fail /dev/sde
```
Проверяем что один диск не активен командой:
```
cat /proc/mdstat
```
Чтобы восстановить рейд необходимо выполнить несколько действий 
1. Удаляем диск который вышел из строя 
```
mdadm /dev/md0 --remove /dev/sde
```
2. Добавляем новый диск взамен старого который вышел из строя 
```
mdadm /dev/md0 --add /dev/sde
```
3. Проверяем что новый диск встал и работает в штатном режиме.
```
cat /proc/mdstat
```
### 5. создать GPT раздел и 5 партиций.

Создаем раздел GPT на RAID
```
parted -s /dev/md0 mklabel gpt
```

Создаем партиции
```
parted /dev/md0 mkpart primary ext4 0% 20%
parted /dev/md0 mkpart primary ext4 20% 40%
parted /dev/md0 mkpart primary ext4 40% 60%
parted /dev/md0 mkpart primary ext4 60% 80%
parted /dev/md0 mkpart primary ext4 80% 100%
```

Далее можно создать на этих партициях ФС
```
for i in $(seq 1 5); do sudo mkfs.ext4 /dev/md0p$i; done
```

И смонтировать их по каталогам
```
mkdir -p /raid/part{1,2,3,4,5}
for i in $(seq 1 5); do mount /dev/md0p$i /raid/part$i; done
```
